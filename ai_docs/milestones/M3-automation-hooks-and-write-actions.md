# M3 - Automation, Hooks & Write Actions

M3 adds safe mutations and automation surfaces once the read model and operator cockpit are stable enough to trust.

## Included epics

- [EPIC-010 - Hooks and Bead write actions](../epics/EPIC-010-hooks-and-bead-write-actions.md)
- [EPIC-011 - Automation surface and mail workflow](../epics/EPIC-011-automation-surface-and-mail-workflow.md)

## Exit criteria

- Gmux can perform the core Gastown hook and Bead write actions defined in this roadmap.
- CLI and socket surfaces cover the main operator flows with machine-readable outputs.
- Mail and inbox flows route users back to the relevant work context.

## Not in scope

- Durable session persistence beyond the minimal needs of the write flows.
- MCP or multi-profile platform work.
